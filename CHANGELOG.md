## v1.4

### Новое: Модульная архитектура LLM адаптеров
- Создана модульная система адаптеров в `backend/llm-adapters/`
- Базовый класс `LLMAdapter` с абстрактными методами:
  - `supports(llmType)` - проверка поддержки типа
  - `call(messages, config)` - вызов LLM API
  - `validateConfig(config)` - валидация конфигурации
  - `createRequestBody(messages, options)` - создание тела запроса
  - `extractResponseText(responseData)` - извлечение текста ответа
- Фабрика `LLMAdapterFactory` для регистрации и выбора адаптеров
- Поддержка 11+ типов LLM: Ollama, LM Studio, Aya, Llama, Mistral, DeepSeek, Qwen, Grok, OpenAI, Anthropic, Google, UncloseAI
- Каждый адаптер в отдельном файле: `ollama.js`, `lmstudio.js`, `openai.js`, `anthropic.js`, `groq.js`, `uncloseai.js`

### Новое: Интеграция пресетов pm-claude-skills
- Клонирован репозиторий pm-claude-skills в `presets/source/`
- Создан конвертер `presets/scripts/convert-skills.js` для преобразования SKILL.md в NeuroOffice роли
- Сгенерировано 95 готовых пресетов в `presets/ready/`
- Категории пресетов: Product Manager (28), Маркетинг (7), Финансы (6), HR (6), Юрист (4), Продажи (4), Операции (4), Инженер (4), Медицина (4), Бизнес (3), Дизайн (3), Мульти (3)
- Русскоязычные имена ролей: "Юрист", "Финансы", "HR", "Продажи", "Маркетинг", "Product Manager"

### Новое: Middleware и утилиты
- Добавлена папка `backend/middleware/` для Express middleware
- Добавлена папка `backend/utils/` для утилит
- Добавлен `backend/cache/` для кэширования

### Новое: Тестирование
- Добавлен `backend/jest.config.js` для unit-тестов
- Добавлены тесты в `backend/__tests__/`
- Скрипт `backend/test-adapters-integration.js` для интеграционного тестирования адаптеров

### Безопасность (из v1.3)
- **API ключи в .env** - токены LLM теперь хранятся в файле `backend/.env`, а не в localStorage браузера
- Бэкенд автоматически читает ключи из переменных окружения (GROQ_API_KEY, OPENAI_API_KEY, ANTHROPIC_API_KEY, GOOGLE_API_KEY)
- UI показывает подсказку о безопасном способе хранения ключей

### Исправления (из v1.3)
- Исправлена синхронизация баз знаний с бэкендом — теперь передаётся полный контент файлов, а не только ID
- Исправлена загрузка пресетов — базы знаний загружаются до ролей
- Исправлено сохранение пресетов — теперь сохраняется полный контент KB
